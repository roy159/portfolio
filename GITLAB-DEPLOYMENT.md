# GitLab Pages Deployment Guide

## How It Works

Your React app **builds to static HTML/CSS/JS** - perfect for GitLab Pages! When you run `pnpm run build`, it creates a `dist` folder with plain HTML files that work without Node.js.

## Quick Deploy to GitLab Pages

### Step 1: Create .gitlab-ci.yml

Create this file in your project root:

```yaml
# .gitlab-ci.yml
image: node:20

pages:
  stage: deploy
  cache:
    key:
      files:
        - pnpm-lock.yaml
    paths:
      - .pnpm-store
  
  before_script:
    - curl -fsSL https://get.pnpm.io/install.sh | sh -
    - export PNPM_HOME="/root/.local/share/pnpm"
    - export PATH="$PNPM_HOME:$PATH"
    - pnpm config set store-dir .pnpm-store
  
  script:
    - pnpm install
    - pnpm run build
    - mv dist public
  
  artifacts:
    paths:
      - public
  
  only:
    - main
```

### Step 2: Update vite.config.ts

If your GitLab repo is **NOT** at `username.gitlab.io`, update the base path:

```typescript
// vite.config.ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  base: '/your-repo-name/',  // ← Change this to your repo name
})
```

**Note:** If your repo IS `username.gitlab.io`, use `base: '/'`

### Step 3: Push to GitLab

```bash
# Initialize git (if not already done)
git init

# Add all files
git add .

# Commit
git commit -m "Initial portfolio commit"

# Add GitLab remote
git remote add origin https://gitlab.com/yourusername/portfolio.git

# Push
git branch -M main
git push -u origin main
```

### Step 4: Enable GitLab Pages

1. Go to your GitLab project
2. Navigate to **Settings** → **Pages**
3. GitLab will automatically detect `.gitlab-ci.yml` and start building
4. Wait 2-3 minutes for the pipeline to complete

### Step 5: Access Your Site

Your site will be live at:
- **username.gitlab.io** (if repo name is `username.gitlab.io`)
- **username.gitlab.io/portfolio** (if repo name is `portfolio`)

## Alternative: Manual Build & Deploy

If you want to build locally and upload:

```bash
# Build the project
pnpm install
pnpm run build

# The 'dist' folder now contains your static HTML/CSS/JS
# Upload the contents of 'dist' folder to GitLab Pages
```

## What Gets Built?

When you run `pnpm run build`, you get:

```
dist/
├── index.html          ← Your main HTML file
├── assets/
│   ├── index-abc123.js     ← Your JavaScript (bundled)
│   ├── index-def456.css    ← Your CSS (bundled)
│   └── ...
└── (any files from /public folder)
```

**This is pure HTML/CSS/JS - no Node.js needed to run it!**


## Troubleshooting

**Pipeline fails?**
- Check `.gitlab-ci.yml` syntax
- View pipeline logs in GitLab: CI/CD → Pipelines

**Page is blank?**
- Check `base` in `vite.config.ts` matches your repo name
- Check browser console for errors

**404 on assets?**
- Verify `base` path in `vite.config.ts`
- Should be `/repo-name/` with trailing slash

## Custom Domain (Optional)

To use a custom domain:

1. In GitLab: **Settings** → **Pages** → Add custom domain
2. Add DNS records at your domain provider:
   - Type: `A`
   - Name: `@`
   - Value: GitLab Pages IP (shown in GitLab)

## Comparison: GitLab vs GitHub Pages

| Feature | GitLab Pages | GitHub Pages |
|---------|--------------|--------------|
| Build Method | CI/CD Pipeline | GitHub Actions |
| Free Tier | ✅ Unlimited | ✅ Unlimited |
| Custom Domain | ✅ Free SSL | ✅ Free SSL |
| Private Repos | ✅ Yes | ❌ Pro only |
